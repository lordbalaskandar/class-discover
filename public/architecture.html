Pulsatract — serverless Lambda architecture

All services run as AWS Lambda functions (Node 20, arm64) deployed via Serverless Framework v4. A single GraphQL gateway fans out to domain-specific Lambdas via Lambda.invoke(). Async work handled by SQS + EventBridge. Full AWS data layer: Aurora PostgreSQL Serverless v2, ElastiCache Redis, Cognito auth, S3 storage.

00 Key architectural decisions

Internal Lambda-to-Lambda invocation. The gateway calls downstream services via Lambda.invoke() in RequestResponse mode — not HTTP. This skips an API Gateway hop on the internal path, reducing latency and cost. Only the gateway and the Stripe webhook Lambda have public HTTP triggers.

Amazon Cognito for auth. Cognito User Pool issues JWTs directly. A dedicated 128 MB Lambda Authorizer verifies tokens via Cognito's JWKS endpoint, queries user_roles in Aurora for the role, and injects { userId, role } into every downstream Lambda's event context. API Gateway caches the result for 300 seconds. Google OAuth wired as a Cognito Identity Provider.

Shared Lambda Layers. The pg driver, stripe SDK, and middy middleware stack are pre-packaged into three layers shared across all functions, reducing cold start time and bundle size. The gateway gets provisioned concurrency (min: 2) to eliminate cold starts on the user-facing path.

Aurora PostgreSQL Serverless v2 + RDS Proxy. All services share one Aurora cluster. RDS Proxy handles connection pooling — essential at Lambda scale where each cold start would otherwise hold a raw connection. PostGIS extension for geo search. pg_trgm for fuzzy text. The cluster auto-scales ACUs from 0.5 to 16, so you pay nothing at zero traffic. IAM database authentication — no password in code.

ElastiCache Serverless (Redis OSS) + S3. ElastiCache Serverless replaces any external Redis provider — it lives in the same VPC, scales automatically, and needs no cluster sizing. Search results cached at 60s TTL, analytics at 5min TTL. S3 + CloudFront handles all media: avatars, class images, gym galleries. Lambdas issue pre-signed upload URLs; clients upload directly to S3.
